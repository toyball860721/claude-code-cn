# 第 6 章：中国特色 — 国内 API 代理、模型替换、网络优化

## 为什么需要这一章？

Claude Code 是 Anthropic 开发的工具，服务器在海外。对于中国大陆的开发者来说，直接使用面临三个核心挑战：

1. **网络连通性**：Anthropic API 在国内无法直连
2. **延迟问题**：即使通过代理，跨太平洋的延迟也会影响体验
3. **成本问题**：海外 API 计费使用美元，汇率和支付方式都是门槛

这一章系统性地解决这三个问题。

## 一、API 代理方案

### 方案概述

API 代理的原理很简单：在你和 Anthropic 之间放一个中转服务，国内请求发到中转服务器，中转服务器再转发到 Anthropic。

```
你的电脑 → API 代理（国内/香港/日本）→ Anthropic API
```

### 配置方法

Claude Code 支持自定义 API 端点：

```bash
# 在 ~/.zshrc 或 ~/.bashrc 中添加
export ANTHROPIC_BASE_URL="https://your-proxy.example.com/v1"
export ANTHROPIC_API_KEY="your-key-from-proxy"
```

### 自建代理

如果你有一台海外 VPS（推荐日本或新加坡，延迟低），可以用 Nginx 做简单的反向代理：

```nginx
# /etc/nginx/conf.d/anthropic-proxy.conf
server {
    listen 443 ssl;
    server_name api-proxy.yourdomain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location /v1/ {
        proxy_pass https://api.anthropic.com/v1/;
        proxy_set_header Host api.anthropic.com;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_ssl_server_name on;

        # 超时设置（Claude 响应可能较慢）
        proxy_read_timeout 300s;
        proxy_connect_timeout 30s;
    }
}
```

自建代理的优势：
- 完全控制数据流向
- 无第三方中间人
- 可以加入缓存层降低成本
- 可以添加速率限制和监控

### 代理选型建议

| 方案 | 延迟 | 成本 | 安全性 | 推荐度 |
|------|------|------|--------|--------|
| 自建代理（日本 VPS）| ~50ms | VPS 月费 | ★★★★★ | ⭐⭐⭐⭐⭐ |
| 第三方 API 中转 | ~100ms | 加价 10-30% | ★★★☆☆ | ⭐⭐⭐ |
| 系统代理（VPN/SSR）| ~200ms | 代理月费 | ★★★★☆ | ⭐⭐⭐⭐ |

## 二、国产模型替代方案

如果你不想折腾代理，或者想降低成本，可以使用国产大模型替代 Claude。虽然能力可能有差异，但对于大部分日常编码任务足够用。

### 方案一：通义千问（Qwen）

阿里通义千问是目前国产模型中编码能力最强的之一：

```bash
# 通过 OpenClaw 使用通义千问
# 在 OpenClaw 配置中设置模型为 modelstudio/qwen3.5-plus
```

优势：
- 国内直连，无需代理
- 中文理解能力极强
- 支付宝直接付费，无汇率问题
- API 价格远低于 Claude

局限：
- 超长上下文能力与 Claude 有差距
- 复杂推理任务略逊色
- Agent 工具调用的稳定性待提升

### 方案二：DeepSeek

DeepSeek 在代码生成方面表现优秀：

```bash
export OPENAI_API_BASE="https://api.deepseek.com/v1"
export OPENAI_API_KEY="your-deepseek-key"
```

### 方案三：混合路由策略（推荐）

最佳实践是混合使用多个模型：

```
日常编码（简单任务）→ Qwen / DeepSeek（国内直连，低成本）
复杂推理（架构设计）→ Claude（通过代理，高质量）
代码审查（中等任务）→ Qwen（性价比高）
```

在 OpenClaw 中，你可以配置模型路由规则，自动根据任务类型选择模型。这样既保证了关键任务的质量，又控制了成本。

## 三、网络优化技巧

### 1. 终端代理配置

让终端流量走代理而不影响其他应用：

```bash
# 在 ~/.zshrc 中添加代理开关函数
proxy_on() {
    export HTTP_PROXY="http://127.0.0.1:7890"
    export HTTPS_PROXY="http://127.0.0.1:7890"
    export ALL_PROXY="socks5://127.0.0.1:7891"
    echo "🌐 代理已开启"
}

proxy_off() {
    unset HTTP_PROXY HTTPS_PROXY ALL_PROXY
    echo "🔒 代理已关闭"
}
```

使用时只需：
```bash
proxy_on   # 开启代理
claude     # 使用 Claude Code
proxy_off  # 关闭代理
```

### 2. npm/pip 镜像配置

安装依赖时使用国内镜像，避免因为代理问题导致安装失败：

```bash
# npm 使用淘宝镜像
npm config set registry https://registry.npmmirror.com

# pip 使用清华镜像
pip config set global.index-url https://pypi.tuna.tsinghua.edu.cn/simple

# Docker 使用阿里云镜像
# 编辑 /etc/docker/daemon.json
{
    "registry-mirrors": ["https://xxxxx.mirror.aliyuncs.com"]
}
```

### 3. Git 加速

```bash
# GitHub 加速
git config --global url."https://ghproxy.com/https://github.com".insteadOf "https://github.com"

# 或者使用 SSH + 代理
# ~/.ssh/config
Host github.com
    ProxyCommand nc -x 127.0.0.1:7891 %h %p
```

### 4. 减少 API 调用次数

每次 API 调用都要经过代理，减少调用次数 = 减少延迟影响：

- **写好 CLAUDE.md**：减少 Claude 需要"探索"项目的次数
- **一次说清需求**：不要碎片化地发指令
- **用 --print 模式**：批量处理减少交互轮次
- **善用 /compact**：压缩上下文减少 token 传输

### 5. 本地缓存策略

对于重复性的查询（如文档检索），可以在本地维护缓存：

```bash
# 将常用文档下载到本地
mkdir -p ~/.claude/docs
# 下载 FastAPI 文档
wget -r -np https://fastapi.tiangolo.com/ -P ~/.claude/docs/fastapi

# 在 CLAUDE.md 中告诉 Claude 优先查本地文档
```

## 四、费用控制

### 监控 API 用量

```bash
# 查看当月 API 用量
curl -s https://api.anthropic.com/v1/usage \
  -H "x-api-key: $ANTHROPIC_API_KEY" | python3 -m json.tool
```

### 预算控制建议

| 使用场景 | 月预算建议 | 对应约 token 数 |
|----------|-----------|----------------|
| 个人轻度使用 | ¥50-100 | ~500 万 tokens |
| 个人重度使用 | ¥200-500 | ~2000 万 tokens |
| 小团队（3-5 人）| ¥500-1500 | ~5000 万 tokens |

### 省钱技巧

1. **简单任务用国产模型**：日常补全、简单修改用 Qwen，省 70% 费用
2. **减少不必要的文件读取**：在 CLAUDE.md 中明确项目结构，避免 AI 扫描整个目录
3. **用 --print 而非交互模式**：减少多轮对话的 token 开销
4. **定期清理 Session**：长 Session 会积累大量 context token

## 五、合规注意事项

- **数据出境**：使用海外 API 时，代码会发送到 Anthropic 的服务器。如果你的项目涉及敏感数据，请评估数据出境风险
- **使用国产模型**：如果合规要求严格，优先使用通义千问、DeepSeek 等国产模型
- **不要发送密钥**：确保 `.env` 文件在 `.gitignore` 和 `.claudeignore` 中
- **企业场景**：建议咨询法务团队关于 AI 辅助编程的合规要求

## 小结

中国开发者使用 Claude Code 的最佳策略是"混合模式"：

- 日常编码用国产模型（零延迟、低成本、合规）
- 关键任务用 Claude（通过自建代理、高质量输出）
- 网络配置做好一劳永逸的优化
- 费用控制精打细算
