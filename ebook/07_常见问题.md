# 第 7 章：常见问题 — 30 个 FAQ

## 安装与配置

### Q1：安装时报错 `npm ERR! code EACCES`，怎么办？

这是权限问题。不要使用 `sudo npm install -g`，改用 nvm 管理 Node.js：

```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
nvm install 22
npm install -g @anthropic-ai/claude-code
```

### Q2：安装成功了但运行 `claude` 报 "command not found"？

检查 npm 全局包的 bin 路径是否在 PATH 中：

```bash
npm config get prefix  # 查看 npm 全局路径
# 将输出路径/bin 添加到 PATH
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

### Q3：API Key 配置了但报 401 错误？

常见原因：
1. Key 格式不对——Anthropic 的 Key 以 `sk-ant-` 开头
2. Key 过期或被吊销——去 console.anthropic.com 重新生成
3. 环境变量没有生效——运行 `echo $ANTHROPIC_API_KEY` 确认

### Q4：国内无法访问 Anthropic API 怎么办？

参见第 6 章"中国特色"，三种方案：
1. 自建 API 代理（推荐）
2. 使用第三方 API 中转服务
3. 改用国产模型（通义千问/DeepSeek）

### Q5：可以离线使用 Claude Code 吗？

不能。Claude Code 需要连接 API 服务来运行模型。但你可以：
- 提前在有网时完成编码工作
- 使用本地模型（如 Ollama + CodeLlama），但体验差距较大

## 日常使用

### Q6：Claude Code 和 Cursor 哪个更好？

没有绝对的"更好"，看你的工作方式：
- **偏好图形界面** → Cursor
- **偏好终端工作流** → Claude Code
- **需要 Agent 自主执行** → Claude Code
- **需要实时代码补全** → Cursor
- 两者可以同时使用，不冲突

### Q7：单次对话最多能处理多少代码？

Claude 的上下文窗口约 200K tokens，大约相当于：
- ~15 万行代码（仅代码，不含对话历史）
- ~50 个中等大小的源文件
- 实际操作中，建议每次聚焦在 5-10 个相关文件

### Q8：Claude Code 会把我的代码发送到外部服务器吗？

是的。你的代码片段会发送到 Anthropic 的 API 服务器进行处理。Anthropic 承诺不使用 API 输入数据来训练模型。如果代码涉及商业机密或敏感信息，请评估风险后决定。使用国产模型可以避免数据出境。

### Q9：如何让 Claude Code 忽略某些文件？

创建 `.claudeignore` 文件（语法同 `.gitignore`）：

```
# .claudeignore
node_modules/
.env
*.log
dist/
coverage/
secrets/
```

### Q10：Session 意外中断了，能恢复吗？

可以：
```bash
claude --resume  # 恢复上一个 Session
```

如果需要恢复更早的 Session，查看 `~/.claude/projects/` 目录下的历史记录。

## 效率与技巧

### Q11：Claude Code 老是读取太多无关文件，怎么控制？

1. 写好 CLAUDE.md，明确项目结构
2. 在指令中指定范围："只看 src/auth/ 目录"
3. 使用 .claudeignore 排除无关目录

### Q12：回复速度太慢怎么办？

1. 检查网络延迟（代理节点是否最优）
2. 简化指令，减少需要分析的文件
3. 用 `--print` 模式替代交互模式
4. 切换到响应更快的模型

### Q13：代码质量不稳定，时好时差？

1. 在 CLAUDE.md 中写明编码规范
2. 使用 Skills 固化最佳实践
3. 要求 Claude 在生成代码后运行 lint 和测试
4. 复杂逻辑要求先写方案再写代码

### Q14：如何让 Claude Code 记住我的偏好？

在 `~/.claude/CLAUDE.md`（全局配置）中写入：

```markdown
## 我的偏好
- 代码注释使用中文
- 变量命名用 camelCase
- 优先使用函数式编程风格
- 测试框架统一使用 Vitest
- CSS 使用 Tailwind
```

### Q15：怎么在团队中共享 Skills？

将 Skills 文件放入项目仓库：

```
your-project/
├── .claude/
│   └── skills/
│       ├── code-review.md
│       └── commit-message.md
└── ...
```

团队成员 clone 项目后，Skills 自动可用。

## 高级功能

### Q16：如何在 CI/CD 中使用 Claude Code？

```yaml
# GitHub Actions 示例
- name: AI Code Review
  run: |
    npm install -g @anthropic-ai/claude-code
    claude --print "审查这个 PR 的代码改动，输出审查报告" \
      --permission-mode bypassPermissions
  env:
    ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }}
```

### Q17：MCP 是什么？怎么用？

MCP（Model Context Protocol）是 Anthropic 推出的开放协议，让 AI 模型能够连接外部工具和数据源。Claude Code 通过 MCP 实现了 Tools 系统。你可以通过 MCP 让 Claude Code 连接数据库、API、文件系统等。

### Q18：能否同时运行多个 Claude Code 实例？

可以。每个终端窗口运行独立的 Session，互不干扰。适合并行处理前端和后端任务。

### Q19：如何限制 Claude Code 的文件访问范围？

通过权限配置：

```json
{
  "permissions": {
    "allow": ["read:src/**", "write:src/**"],
    "deny": ["read:.env", "write:config/production.*"]
  }
}
```

### Q20：子 Agent 是什么？什么时候用？

子 Agent 是主 Agent 派生的独立任务执行者。适合将一个大任务分解为多个并行子任务。例如主 Agent 负责协调，子 Agent A 负责前端、子 Agent B 负责后端。

## 故障排除

### Q21：报错 "context window exceeded"？

上下文超限了。解决方法：
1. 运行 `/compact` 压缩上下文
2. 开一个新 Session，只加载必要的文件
3. 减少单次读取的文件数量

### Q22：Claude Code 修改了文件但结果不对，怎么回退？

如果在 Git 仓库中：
```bash
git checkout -- path/to/file  # 回退单个文件
git stash                      # 暂存所有改动
git reset --hard HEAD          # 回退到上次 commit
```

建议在让 Claude Code 大规模修改代码之前先 `git commit` 一次。

### Q23：权限弹窗太频繁，影响效率？

配置自动批准常用操作：

```json
{
  "permissions": {
    "allow": [
      "read:**",
      "exec:npm test",
      "exec:npm run *",
      "exec:git *"
    ]
  }
}
```

### Q24：Claude Code 生成的代码有 TypeScript 类型错误？

1. 确保 CLAUDE.md 中声明了 TypeScript 版本和 tsconfig 配置
2. 让 Claude 在修改后运行 `npx tsc --noEmit` 检查类型
3. 如果用了最新的 TS 特性，告诉 Claude 你的 TS 版本

### Q25：网络断开后 Session 怎么办？

Session 状态保存在本地。网络恢复后运行 `claude --resume` 即可继续。未完成的操作可能需要重新执行。

## 成本与计费

### Q26：Claude Code 本身要钱吗？

Claude Code CLI 工具本身是免费的。但使用它需要 Anthropic API Key，API 调用按 token 计费。定价参见 Anthropic 官网。也可以通过 Claude Pro/Max 订阅获得包含的额度。

### Q27：一天大概花多少钱？

取决于使用强度：
- 轻度使用（5-10 次交互）：$1-3/天
- 中度使用（20-50 次交互）：$5-15/天
- 重度使用（持续编码）：$20-50/天
- 混合策略（简单任务用国产模型）：可节省 50-70%

### Q28：怎么监控费用？

1. Anthropic Console 有用量面板
2. 在 OpenClaw 中配置预算控制
3. 设置每日预算上限，超限自动停止

### Q29：团队使用怎么计费？

建议每人使用独立的 API Key，通过 Anthropic 的 Organization 功能管理。这样可以追踪每个人的用量，设置独立的预算限制。
