# 第 1 章：安装配置 — 5 分钟上手

## 系统要求

在开始之前，确认你的环境满足以下条件：

- **操作系统**：macOS 12+、Ubuntu 20.04+、Windows 11（WSL2）
- **Node.js**：v18 或更高版本（推荐 v22 LTS）
- **终端**：任何支持 ANSI 的终端（推荐 iTerm2 / Warp / Windows Terminal）
- **网络**：能访问 Anthropic API（国内用户见下方代理配置）

## 第一步：安装 Node.js

如果你还没有 Node.js，最快的方式是用 nvm：

```bash
# macOS / Linux
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.0/install.sh | bash
source ~/.zshrc  # 或 source ~/.bashrc
nvm install 22
nvm use 22

# 验证
node -v  # 应该显示 v22.x.x
```

Windows 用户推荐先安装 WSL2，然后在 WSL 中执行上述命令。原生 Windows 也可以从 [nodejs.org](https://nodejs.org) 下载安装包。

## 第二步：安装 Claude Code

```bash
# 全局安装
npm install -g @anthropic-ai/claude-code

# 验证安装
claude --version
```

如果你在国内遇到 npm 下载慢的问题：

```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com
npm install -g @anthropic-ai/claude-code

# 或使用 cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install -g @anthropic-ai/claude-code
```

## 第三步：配置 API 密钥

Claude Code 需要 Anthropic API 密钥才能工作。获取方式：

1. 访问 [console.anthropic.com](https://console.anthropic.com)
2. 注册账号并创建 API Key
3. 配置环境变量：

```bash
# 方式一：直接设置环境变量
export ANTHROPIC_API_KEY="sk-ant-xxxxx"

# 方式二：写入 shell 配置文件（推荐）
echo 'export ANTHROPIC_API_KEY="sk-ant-xxxxx"' >> ~/.zshrc
source ~/.zshrc

# 方式三：使用 .env 文件（项目级别）
echo 'ANTHROPIC_API_KEY=sk-ant-xxxxx' > .env
```

## 第四步：国内网络配置（重要！）

这是中国开发者最关心的部分。以下提供几种可行方案：

### 方案一：API 代理转发（推荐）

很多国内服务商提供 Anthropic API 的代理转发服务。配置方式：

```bash
# 设置自定义 API 端点
export ANTHROPIC_BASE_URL="https://your-proxy.example.com/v1"
export ANTHROPIC_API_KEY="your-key"
```

常见的代理服务包括但不限于各类 API 中转平台，请根据实际情况选择可靠的服务商。

### 方案二：系统代理

如果你已经有科学上网工具：

```bash
# HTTP 代理
export HTTP_PROXY="http://127.0.0.1:7890"
export HTTPS_PROXY="http://127.0.0.1:7890"

# SOCKS5 代理
export ALL_PROXY="socks5://127.0.0.1:7891"
```

将上述配置写入 `~/.zshrc` 以持久化。

### 方案三：使用兼容的国产模型

Claude Code 支持通过 OpenAI 兼容接口使用其他模型。例如使用阿里通义千问：

```bash
# 通过 OpenClaw 配置模型路由
# 在 openclaw 配置中设置 modelstudio/qwen3.5-plus
```

这种方案完全不需要海外网络，但部分高级功能可能受限于模型能力差异。

## 第五步：首次启动

一切配置完成后，在任意项目目录中启动 Claude Code：

```bash
cd your-project
claude
```

你会看到一个交互式终端界面。试着输入你的第一个指令：

```
> 分析这个项目的结构，告诉我它是做什么的
```

Claude Code 会自动扫描项目文件、读取 README 和配置文件，然后给你一份清晰的项目概述。

## 常用启动参数

```bash
# 以打印模式运行（非交互，适合脚本）
claude --print "帮我找出这个项目中所有的 TODO 注释"

# 指定权限模式
claude --permission-mode interactive  # 每次操作都询问
claude --permission-mode bypassPermissions  # 跳过权限确认（慎用）

# 继续上一个 session
claude --resume
```

## 验证安装成功

运行以下命令确认一切正常：

```bash
# 快速测试：让 Claude Code 生成一个文件
claude --print "创建一个 hello.py，内容为打印 Hello World"

# 如果成功生成了 hello.py，说明安装和 API 配置都没问题
python3 hello.py  # 应该输出 Hello World
```

## 配置文件详解

Claude Code 的配置文件位于 `~/.claude/` 目录下：

```
~/.claude/
├── settings.json    # 全局设置
├── credentials.json # API 密钥（加密存储）
└── projects/        # 项目级别配置缓存
```

你也可以在项目根目录创建 `.claude/` 目录来存放项目级别的配置：

```
your-project/
├── .claude/
│   └── settings.json  # 项目级设置（覆盖全局）
├── CLAUDE.md          # Claude Code 的项目指南文件
└── ...
```

其中 `CLAUDE.md` 是一个特殊文件——Claude Code 每次启动时都会自动读取它，作为理解项目的上下文。我们在第 2 章会详细讲解。
