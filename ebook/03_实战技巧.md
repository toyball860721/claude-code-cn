# 第 3 章：实战技巧 — 20 个让效率翻倍的技巧

## 技巧总览

这 20 个技巧来自真实开发场景的总结，每个都附有可直接使用的示例。按难度递进排列：前 7 个是基础技巧，中间 7 个是进阶技巧，最后 6 个是高级技巧。

---

## 基础篇

### 技巧 1：用自然语言描述需求，而非代码指令

**错误示范：**
```
> 创建一个函数，接收 string 参数 name，返回 "Hello, " + name
```

**正确示范：**
```
> 我需要一个打招呼的功能，传入用户名，返回问候语。要有类型注解和文档字符串。
```

Claude Code 理解意图比理解伪代码更好。告诉它"做什么"而不是"怎么做"。

### 技巧 2：让 Claude 先分析再动手

```
> 先分析 src/services/ 目录下所有文件的职责，列出依赖关系，然后再帮我重构
```

加一个"先分析"能避免 AI 盲目修改。它会先读取文件、理解结构，再制定修改计划。

### 技巧 3：用 CLAUDE.md 固定项目规范

在项目根目录创建 `CLAUDE.md`：

```markdown
## 编码规范
- TypeScript strict mode
- 函数名用 camelCase，类名用 PascalCase
- 所有 API 返回统一格式：{ code, data, message }
- 错误处理使用自定义 AppError 类

## 禁止事项
- 不要使用 any 类型
- 不要使用 console.log（使用项目的 logger）
- 不要硬编码配置值
```

这样每次启动 Claude Code，它都会遵守这些规范。

### 技巧 4：批量操作用 --print 模式

```bash
# 批量给所有 Python 文件添加类型注解
claude --print "给 src/ 下所有 .py 文件的函数添加完整的类型注解" \
  --permission-mode bypassPermissions

# 批量生成测试文件
claude --print "为 src/services/ 下的每个文件生成对应的测试文件到 tests/"
```

`--print` 模式不进入交互界面，执行完就退出，适合批量处理和脚本集成。

### 技巧 5：用 /compact 管理长对话

在一个长 Session 中，上下文会逐渐膨胀。当你感觉 Claude Code 的回复质量下降（开始"忘记"之前的内容），手动压缩：

```
> /compact
```

或者给它一个压缩提示：

```
> /compact 保留：我们在做用户模块重构，已完成 model 层，正在做 service 层
```

### 技巧 6：善用 git 集成

```
> 查看最近 5 次 commit 的改动，总结每次都改了什么
> 帮我 rebase 最近 3 个 commit，合并成一个，写一个清晰的 commit message
> 创建一个新分支 feature/auth，从 main 分支上切出来
```

Claude Code 可以直接操作 git，省去你记忆各种 git 命令的负担。

### 技巧 7：让它解释现有代码

```
> 解释 src/core/scheduler.ts 的核心逻辑，画一个调用流程
> 这个正则表达式是什么意思：/^(?=.*[A-Z])(?=.*\d)[A-Za-z\d@$!%*?&]{8,}$/
> 这段代码的时间复杂度是多少？有没有优化空间？
```

## 进阶篇

### 技巧 8：链式任务指令

把多个相关任务串在一起：

```
> 1. 读取 src/api/routes.ts
> 2. 找到所有没有错误处理的路由
> 3. 给每个路由添加 try-catch 和统一的错误响应
> 4. 运行 npm test 确认没有破坏现有功能
```

Claude Code 会按步骤执行，上一步的结果自动成为下一步的上下文。

### 技巧 9：用对比法做技术选型

```
> 对比 Prisma 和 Drizzle ORM 在以下方面的差异：
> - TypeScript 类型安全
> - 查询性能
> - 学习曲线
> - 社区生态
> 基于我们项目的情况（CLAUDE.md 里描述的），推荐哪个？
```

### 技巧 10：让 Claude 写测试来验证自己的代码

```
> 实现一个 LRU Cache 类，然后为它写完整的单元测试，最后运行测试确保全部通过
```

这个技巧的妙处在于：让 AI 自己写测试来验证自己的实现，形成自我校验闭环。

### 技巧 11：调试错误的正确姿势

**不要这样：** 复制错误信息粘贴给 Claude

**要这样：**
```
> 运行 npm test，如果有失败的测试，分析原因并修复，然后重新运行直到全部通过
```

让 Claude Code 自己看到错误输出，它能获取完整的上下文（堆栈追踪、相关文件），比你手动复制粘贴信息更全。

### 技巧 12：用临时文件做草稿

```
> 在 /tmp/draft.md 里写一份 API 设计方案，列出所有端点。
> 等我确认后再开始写代码。
```

先让 AI 输出计划到临时文件，审核通过后再执行，避免直接修改项目代码。

### 技巧 13：多文件联动修改

```
> 把 User 模型的 email 字段从可选改为必填。
> 请同时更新：模型定义、数据库迁移、API schema、前端表单验证、相关测试。
```

这是 Claude Code 的强项——它能追踪字段在所有文件中的引用，一次性全部更新。

### 技巧 14：代码迁移和升级

```
> 把项目从 Vue 2 Options API 迁移到 Vue 3 Composition API。
> 从 src/components/ 开始，一个文件一个文件地迁移。
> 每迁移一个文件就运行测试确认没有问题。
```

## 高级篇

### 技巧 15：利用 exec 做数据分析

```
> 分析 access.log 中最近 24 小时的请求，找出：
> 1. 请求量最大的 10 个端点
> 2. 平均响应时间最长的 5 个端点
> 3. 5xx 错误率的变化趋势
> 用 awk/sort/uniq 处理，结果输出为 markdown 表格
```

### 技巧 16：自动生成 Commit + PR

```
> 查看当前所有改动（git diff），生成一个符合 Conventional Commits 规范的 commit message。
> 然后创建一个 PR，标题和描述要清楚说明改了什么、为什么改。
```

### 技巧 17：项目健康检查

```
> 对这个项目做一次全面健康检查：
> - 是否有安全漏洞（npm audit）
> - 是否有过时的依赖
> - 代码中是否有明显的 code smell
> - 测试覆盖率是否达标
> - 是否有 TODO/FIXME/HACK 注释需要处理
> 输出一份报告，按优先级排序
```

### 技巧 18：用 Claude Code 写 Claude Code 的 Skills

没错，你可以让 Claude Code 帮你写 Skills：

```
> 帮我写一个 Skill，功能是：当我说"优化 SQL"时，自动分析我给出的 SQL 语句，
> 给出索引建议、查询改写方案和 EXPLAIN 分析。输出为 SKILL.md 格式。
```

### 技巧 19：并行任务处理

对于独立的任务，可以开多个终端窗口分别运行 Claude Code：

```bash
# 终端 1：处理前端
cd frontend && claude --print "优化所有组件的 re-render 性能"

# 终端 2：处理后端
cd backend && claude --print "给所有 API 添加请求验证"

# 终端 3：处理基础设施
cd infra && claude --print "更新 Kubernetes 配置，添加资源限制"
```

### 技巧 20：构建自动化流水线

将 Claude Code 集成到你的开发流程中：

```bash
#!/bin/bash
# pre-commit hook
claude --print "检查暂存的文件是否有明显的 bug 或安全问题。只输出问题，没有问题就输出 OK" \
  --permission-mode bypassPermissions

# 如果 Claude 发现问题，阻止 commit
if [[ $? -ne 0 ]]; then
  echo "Claude Code 发现了问题，请修复后再提交"
  exit 1
fi
```

## 小结

这 20 个技巧覆盖了从基础使用到高级集成的完整路径。最重要的三个原则：

1. **告诉 AI 目标，而不是步骤**——让它发挥 Agent 的自主性
2. **先分析后执行**——避免盲目修改
3. **让 AI 自我验证**——写代码后跑测试，形成闭环
