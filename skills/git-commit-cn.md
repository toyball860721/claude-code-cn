# 中文 Git Commit 生成器

> 分析暂存的代码变更，自动生成符合 Conventional Commits 规范的中文 git commit message。

## 触发条件

当用户说以下内容时触发：
- "提交代码"
- "生成 commit"
- "写 commit message"
- "中文提交"

## 执行步骤

1. **检查暂存区**：运行 `git diff --staged` 查看暂存的改动
   - 如果暂存区为空，运行 `git diff` 查看工作区改动
   - 如果工作区也没有改动，提示用户"没有检测到代码改动"
   - 如果有工作区改动但未暂存，提示用户先执行 `git add`

2. **分析改动**：
   - 统计改动文件数量和类型
   - 识别改动的主要范围（哪个模块/功能）
   - 判断改动类型（新增功能、修复 bug、重构、文档等）

3. **生成 Commit Message**：按以下格式输出

## 输出格式

```
<类型>(<范围>): <简短描述>

<正文：详细说明改动内容>

<关联信息（可选）>
```

### 类型映射表

| 类型 | 中文说明 | 使用场景 |
|------|---------|---------|
| feat | 新功能 | 添加新功能或特性 |
| fix | 修复 | 修复 bug |
| docs | 文档 | 文档更新 |
| style | 格式 | 代码格式化，不影响逻辑 |
| refactor | 重构 | 代码重构，不改变功能 |
| perf | 性能 | 性能优化 |
| test | 测试 | 添加或修改测试 |
| build | 构建 | 构建系统或依赖变更 |
| ci | CI | CI 配置变更 |
| chore | 杂项 | 其他不影响代码的改动 |

### 示例输出

```
feat(用户模块): 添加邮箱验证功能

- 新增 EmailVerification 模型，支持验证码 6 位数字
- 添加发送验证邮件的 API 端点 POST /api/verify-email
- 集成阿里云邮件推送服务
- 验证码有效期设为 15 分钟

关联: #123
```

## 规则

- 简短描述不超过 50 个字符
- 使用中文撰写正文
- 如果改动涉及多个范围，用逗号分隔：`feat(用户,订单): ...`
- 破坏性变更在正文中用 `⚠️ BREAKING CHANGE:` 标注
- 如果有关联的 Issue，在最后一行添加 `关联: #xxx`

## 执行后动作

生成 commit message 后，询问用户：
1. 直接提交（`git commit -m "..."`）
2. 修改后提交
3. 取消

## 注意事项

- 不自动执行 `git add`，暂存操作应由用户控制
- 如果改动过大（超过 500 行），建议用户拆分 commit
- 合并 commit 时使用 `git commit --amend` 参数
